# T17: Ruff - CLI Config Overrides (`--config key=value`)

*Upstream reference:* https://github.com/astral-sh/ruff/issues/8368 — full specification inlined below so no internet access is needed.

## 1. Requirement

Ruff supports two modes of configuration today:

1. A TOML file (`ruff.toml`, `pyproject.toml`, or an explicit `--config path/to/file.toml`) discovered from the filesystem and merged with user defaults.
2. Command-line flags for a small set of high-value options (`--line-length`, `--select`, `--ignore`, `--extend-select`, etc).

This task adds a third mechanism: **key/value overrides on the command line**. A user should be able to do

```
ruff check --config 'line-length=110' --config 'lint.select=["E","F","I"]' .
```

and have those values take the same precedence as a config file but override settings from any discovered file. Each `--config` flag may still point at a path; the value form is recognised when it contains a TOML key=value.

## 2. Motivation

Users repeatedly request ad-hoc configuration overrides at the CLI without having to edit their `pyproject.toml` or maintain extra files for CI. The existing design exposes only a hand-picked subset as CLI flags, so rarely-tuned options (e.g. `per-file-ignores`, new plugin knobs) are unreachable from the command line.

The new feature lets users set any valid TOML configuration key exactly as they would in a config file, in keeping with Ruff's goal of being a single self-contained binary.

## 3. Detailed design

### 3.1 CLI surface
`--config` may now appear multiple times. Each occurrence is one of:

- A path to an existing configuration file (`ruff.toml`, `pyproject.toml`, or an arbitrary `.toml` file containing `[tool.ruff]`).
- A TOML key=value string such as `line-length=120` or `lint.select=["E","F"]`. The string must be parseable as a single TOML table entry.

Overrides apply *after* the auto-discovered configuration: each override overwrites or merges with the values from discovered files in the order given.

### 3.2 Parsing
A small TOML parser reads each override into a `toml::Value`. Bare identifiers such as `line-length` are interpreted as dotted keys under `[tool.ruff]`. Nested tables are expressed with dotted paths:

```text
--config "lint.select=['E','F']"
--config "lint.per-file-ignores={'tests/*'=['D'], 'src/*'=['E501']}"
```

If parsing fails we report a precise message pointing at the offending `--config` argument.

### 3.3 Merge order

After discovery and prior to pyproject/tool resolution the settings resolution order is:

1. `--config path.toml` entries in CLI order.
2. Auto-discovered `pyproject.toml`.
3. Auto-discovered `ruff.toml`.
4. `--config key=value` entries, layered in CLI order (later ones override earlier).
5. Explicit CLI flags (unchanged semantics).

### 3.4 Option table

All keys recognisable inside a `ruff.toml`/`pyproject.toml` must be accepted by `--config key=value`. Keys are normalised to their canonical form (dashes become underscores).

### 3.5 Error handling

- Invalid key: "unknown configuration key 'xyz' at --config".
- Invalid value: "invalid value for 'line_length': expected integer".

## 4. Implementation Notes

1. Extend the Ruff CLI argument struct so `--config` accepts an arbitrary number of strings and preserves their order.
2. In the workspace configuration module, add a layering pass that parses each `key=value` argument into a TOML fragment and merges it into the layered config.
3. Document the new syntax in the user-facing configuration documentation.
4. Update the autogenerated Ruff configuration JSON schema.
5. Add snapshots in the Ruff CLI integration tests exercising the new form.

## 5. Acceptance Criteria

- `ruff check --config 'select=["E"]'` succeeds and lints only E-prefixed rules.
- `ruff check --config 'line-length=120' --config 'select=["E501"]'` layers both overrides correctly.
- An existing `pyproject.toml` / `ruff.toml` continues to drive the base configuration; `--config` overrides compose on top.
- An unknown key produces a clear error including the offending CLI argument.
